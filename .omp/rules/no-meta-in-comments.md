注释只描述代码**当前的行为、不变式与取舍**，不携带设计过程信息：

- 禁止引用 issue / PR 编号（如「issue #29」「见 PR #12」）——编号随仓库迁移失效，且注释读者不需要考古；背景放 commit message。
- 禁止「对标 X」「参考自 X」「借鉴 X」类出处说明——写清楚**本代码做什么、为什么**即可，无需援引外部项目背书。
- 禁止复述需求文档/讨论结论（「硬要求」「用户期望」等措辞）——改写为代码自身的契约陈述。
- 允许：`BUG-XXX` 内部缺陷标签、RFC/标准编号（如 RFC 7233）、算法学名——这些是稳定的技术引用，不是过程信息。

修正方式：删掉编号/出处，保留（或改写出）行为与原因本身。
